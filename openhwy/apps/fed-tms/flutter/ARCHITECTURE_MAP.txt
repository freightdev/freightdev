┌──────────────────────────────────────────────────────────────────────────────┐
│                         COMPLETE DATA FLOW                                   │
│                  Example: User Starts an Agent                               │
└──────────────────────────────────────────────────────────────────────────────┘

1. User Action (Flutter):
   User taps "Start" on AgentCard for HWY agent
   
2. Frontend → Backend:
   POST http://workbox:8080/api/agents/launch
   Body: {
     "system_name": "workbox",
     "agent_type": "hwy",
     "config_path": "/home/admin/agents/hwy.toml",
     "script_path": "/home/admin/agents/hwy.lua"
   }

3. Rust Backend:
   ├─→ Receives request
   ├─→ SSH to workbox: ssh admin@workbox
   ├─→ Execute: cd ~/agents && moon-env hwy.toml hwy.lua &
   ├─→ Capture PID
   └─→ Return: { "success": true, "pid": 12345 }

4. Moon Environment (on workbox):
   ├─→ Loads hwy.toml (config)
   ├─→ Loads hwy.lua (agent code)
   ├─→ Creates sandbox
   ├─→ Starts agent
   └─→ Agent begins execution

5. Agent Startup:
   ├─→ HWY connects to Rust backend
   ├─→ Registers via: POST localhost:8080/internal/agent/register
   ├─→ Opens WebSocket: ws://localhost:8080/ws/agent
   └─→ Sends heartbeat every 30s

6. Backend Updates:
   ├─→ Updates system stats
   ├─→ Broadcasts via WebSocket:
       {
         "type": "agent_update",
         "data": {
           "name": "HWY",
           "status": "running",
           "system_name": "workbox",
           ...
         }
       }

7. Frontend Receives Update:
   ├─→ WebSocketService receives message
   ├─→ Parses agent_update event
   ├─→ Updates AgentModel in state
   └─→ AgentCard automatically re-renders (shows green status)


┌──────────────────────────────────────────────────────────────────────────────┐
│                         PROTOBUF INTEGRATION                                 │
│                  Type-Safe Communication Layer                               │
└──────────────────────────────────────────────────────────────────────────────┘

openhwy.proto defines all messages:

Backend (Rust):
  ├─→ Compile: protoc --rust_out=src proto/openhwy.proto
  ├─→ Import: use openhwy_proto::System;
  └─→ Serialize: system.encode_to_vec()

Backend (Go):
  ├─→ Compile: protoc --go_out=. proto/openhwy.proto
  ├─→ Import: import pb "openhwy/proto"
  └─→ Serialize: proto.Marshal(&system)

Frontend (Flutter):
  ├─→ Compile: protoc --dart_out=lib/proto proto/openhwy.proto
  ├─→ Import: import 'package:openhwy_dashboard/proto/openhwy.pb.dart';
  └─→ Deserialize: System.fromBuffer(bytes)

For JSON REST API:
  ├─→ Backend serializes protobuf → JSON
  └─→ Flutter deserializes JSON → Model

For gRPC:
  ├─→ Direct protobuf binary (more efficient)
  └─→ Flutter gRPC client → Backend gRPC server
