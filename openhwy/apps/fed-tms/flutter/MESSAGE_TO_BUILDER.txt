═══════════════════════════════════════════════════════════════════════════════
                         COMMUNICATION PROTOCOL
═══════════════════════════════════════════════════════════════════════════════

Agents talk via: MessagePack (binary) or Protocol Buffers

Why binary?
  - Fast serialization
  - Small payload size
  - Type-safe
  - Cross-language

Message structure (.proto):
  ```protobuf
  syntax = "proto3";
  
  message AgentRequest {
    string request_id = 1;
    string from_agent = 2;
    string to_agent = 3;
    string action = 4;
    bytes payload = 5;
    int64 timestamp = 6;
  }
  
  message AgentResponse {
    string request_id = 1;
    bool success = 2;
    bytes result = 3;
    string error = 4;
    int64 timestamp = 5;
  }
  
  message CommandRequest {
    string command = 1;
    repeated string args = 2;
    map<string, string> env = 3;
    string working_dir = 4;
  }
  
  message CommandResponse {
    int32 exit_code = 1;
    string stdout = 2;
    string stderr = 3;
    int64 execution_time_ms = 4;
  }
  ```

Compile these:
  ```bash
  protoc --rust_out=src/proto types/agency.proto
  protoc --python_out=py/proto types/agency.proto
  ```

═══════════════════════════════════════════════════════════════════════════════
                         AGENT RULES (what to read on startup)
═══════════════════════════════════════════════════════════════════════════════

Each agent has: config.toml

Example: agents/data_collector/config.toml

```toml
[agent]
name = "data_collector"
version = "1.0.0"

[capabilities]
web_search = true
web_fetch = true
storage_write = true
storage_read = false

[rules]
# What I can do
allowed_domains = ["*"]  # All domains by default
blocked_domains = ["facebook.com", "twitter.com"]  # Social media (too much noise)
max_concurrent_requests = 10
timeout_seconds = 30

# What I must do
log_all_requests = true
cache_results = true
rate_limit_per_domain = "10 req/min"

[tools]
# My internal tools (in tools/ directory)
internal = ["web_search.py", "scraper.py", "parser.py"]

# External tools I can request from COMMAND COORDINATOR
external = ["curl", "wget", "jq"]

[communication]
protocol = "protobuf"  # or "messagepack"
coordinator_address = "localhost:50051"

[routing]
# Where to send specific requests
database_writes = "database_manager"
file_storage = "storage_service"
notifications = "notification_service"
```

Agent startup sequence:
1. Read config.toml
2. Load rules (what I can/can't do)
3. Initialize tools (internal)
4. Connect to COMMAND COORDINATOR
5. Request keys
6. Ready to receive work


═══════════════════════════════════════════════════════════════════════════════
                         TOOLS vs AGENTS
═══════════════════════════════════════════════════════════════════════════════

TOOLS are inside agent directories:
  agents/data_collector/tools/web_search.py
  
  The agent OWNS these tools.
  The agent can run them directly.
  No permission needed.

EXTERNAL TOOLS require coordinator:
  Agent: "I need to run: curl https://example.com"
  Coordinator: "Here's your key, execute via service_manager"
  Service Manager: Executes curl, returns output
  Agent: Receives output, continues work

WHEELERS are special tools:
  Wheeler = Full agent with sub-tools (wheels)
  Example: packet_pilot wheeler
    - Has wheel: ocr/
    - Has wheel: form_filler/
    - Has wheel: signature/
  
  Wheeler is spawned by COMMAND COORDINATOR when needed
  Wheeler runs in Firecracker microVM (isolated)
  Wheeler communicates via same protocol (protobuf/messagepack)
